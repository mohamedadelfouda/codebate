## سلطة المستندات (Document authority)

> **هذه الخريطة هي المصدر الرسمي للأولوية والترتيب والاعتماديات فقط.**
> المستندات المصدرية الأربعة تظل **المرجع الرسمي للتفاصيل التقنية** — معايير القبول،
> عقود البيانات، اختبارات الأمان، حالات الفشل، وتفاصيل التنفيذ — ما لم تنص الخريطة صراحةً
> على استبدال بند منها.
>
> باختصار: **الخريطة تقول "متى ولماذا"، والخطط تقول "كيف ومتى نعتبر البند منتهيًا".**
> لا تُحذَف الخطط الأربع بعد الدمج.

## Alpha launch gate

لا يتم إطلاق ألفا إلا بعد:

- [ ] P0-1 / P0-2 / P0-3 مكتملة واختباراتها الخاصة خضراء.
- [ ] Setup Doctor ينجح مع صفر / واحد / اثنين من المزوّدين.
- [ ] Source Preflight يعمل على Windows / macOS / Linux.
- [ ] الحفظ المتين للحالات الطرفية (P1-1) والاسترداد الأساسي مكتملان.
- [ ] Usage instrumentation (P1-7) يسجّل `real` / `unavailable` بصدق.
- [ ] Mini-eval بالأذرع الأربعة (S-2) مكتمل ونتائجه منشورة.
- [ ] لا توجد ملاحظة Critical أو High **صالحة وغير محلولة** ضمن نطاق الإصدار. أي ملاحظة
      مستبعدة تحمل سببًا موثقًا، وأي مخاطرة مقبولة تحمل قرار قبول صريحًا من المالك (Findings
      كاذبة لا توقف الإطلاق، وHigh حقيقية لا تُتجاهَل بصمت).
- [ ] Clean-clone smoke test موثّق ويمرّ.
- [ ] Version + Tag + Changelog + GitHub Release جاهزة (SD-4a).

> **Alpha gate ≠ Stable gate.** الإصدار المستقر لاحقًا يحتاج مدة استخدام حقيقية،
> بيانات تعافٍ، واختبارات أوسع — بوابة منفصلة تُعرّف عند الاقتراب منه.

## المبدأ التوجيهي

> **قياس الاستخدام (P1-7) + الـ benchmark النصي (S-2) قبل أي ميزة جديدة أو مزوّد ثالث** —
> عشان كل إضافة (مرفقات، Cursor، تتبّع الحصة) تُقاس أثرها الفعلي بدل ما تتخلط مع بعض.

أكبر خطر مذكور في `PROJECT_REVIEW.md` هو **"قيمة غير مُثبَتة"** — مش نقص ميزات. لذلك:
1. **P0 أولاً** — بدون هذا، أي مستخدم (قديم أو جديد) يفقد الثقة فوراً.
2. **Setup Doctor ثانياً** — التقرير نفسه يقول: *"الخطر الحقيقي هو التوزيع، مش الجودة."*
3. **إثبات القيمة قبل أي توسّع** — إضافة Cursor أو المرفقات قبل الـ benchmark بناء على افتراض، لا دليل.
