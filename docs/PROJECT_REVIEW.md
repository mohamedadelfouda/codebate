# Codebate — تقرير التقييم الشامل

> **التاريخ:** 2026-07-16 · **الحالة المُراجَعة:** `main` بعد دمج PR #22
> **المنهجية:** مراجعة كود مباشرة لكل الطبقات + مستشارَي Fable 5 (منتج/سوق + معمار/إطلاق) + استكشاف خط التنفيذ والواجهة. الأدلة بصيغة `ملف:سطر`.

---

## ⭐ الملخّص التنفيذي

المشروع **هندسته أعلى بكتير من متوسط أي مشروع أوبن سورس فردي**: طبقات حقيقية، نواة حتمية نقية، اختبارات crash-matrix على 3 أنظمة، صفر تبعيات runtime، وتعليقات أمينة بتسمّي مخاطرها بنفسها.

**الإجابة على السؤال المحوري ("هل أقدّم جديد؟"):**

> **نعم، بس في آلية واحدة عميقة — مش في الفئة كلها.** الجديد الحقيقي والدفاعي هو **السجل الحتمي بالإجماع** (`server/convergence.js`) اللي بيمنع الوكلاء يقفلوا بند **بالصمت** أو من طرف واحد، وبيثبّت النتيجة **قبل** ما الـ LLM يتكلم. التأطير الأدق: **"محرّك محاسب، مش قاضي — بيرفض إن الوكلاء يمثّلوا."** لكن عنوان *"وكيلين بيتناقشوا للتقارب"* مش جديد وهو **أضعف نص** في المنتج (السوق راح لـ selection + review bots).

**أكبر 3 حاجات لازم تتعالج قبل الإطلاق:**
1. زر **Stop التنفيذ مابيوقفش الكاتب فعلاً**.
2. الواجهة **مابترجعش لحالتها بعد انقطاع SSE** (بتفضل "running").
3. **اللغة الافتراضية عربي + نصوص سيرفر عربي hardcoded** هتبان "مكسورة" لجمهور الإنجليزي.

## المحور 1 — المنتج والسوق

### 1.1 إيه هو فعلاً
سيرفر **محلي loopback**، وكيلان (Claude Code + Codex) بياخدوا نفس حزمة الأدلة، واحد يقترح وواحد يراجع read-only، و**طبقة حتمية (مش الـ LLM)** هي السلطة اللي بتسجّل الاتفاق/الخلاف في سجل بنود مُتحقَّق آلياً، وكل تغيير أو side-effect وراء قرار مستخدم صريح (Execute→Review→Decide).

### 1.2 هل فيه جديد؟ — مقارنة بالسوق 2026

| الأداة | بتعمل إيه | Codebate بيفرق فين |
|---|---|---|
| **Cursor 2.0** | وكلاء متعددين بالتوازي → **اختَر الأحسن** | مفيش حالة اتفاق مُتحقَّقة؛ Codebate *تقارب* مش *اختيار* |
| **aider** | architect/editor split | aider بيعمل commit تلقائي بدون بروتوكول اتفاق؛ Codebate العكس تماماً |
| **Cline/Roo** | موافقة بشرية لكل خطوة | بوابة القرار لوحدها **مش جديدة**؛ بس مفيش عندهم convergence state |
| **Claude Code/Codex نفسهم** | الأساس اللي المشروع بيركب عليه | **أكبر خطر ابتلاع** — لو أي vendor شحن "خلّي الـ CLI التاني ينقد خطتك" ياخد 70% من القيمة |
| **GitHub Agent HQ** (أكت 2025) | mission control لعدة vendors + سلطة merge بشرية | النسخة المنصّية من نفس الفكرة |
| **CrewAI/AutoGen/LangGraph** | frameworks | تقدر تبني نفس الشكل، بس الإنهاء string-match أو LLM-judge؛ محدش بيديك **سجل بنود مُتحقَّق بالإجماع** |
| **CodeRabbit/Greptile** | مراجعة **بعد** الـ push، سحابية | Codebate **قبل** الـ commit، محلي، خاص — **تفريق حقيقي** |

**اللي محدش بيشحنه فعلاً:**
1. **مُجمِّع حتمي كسلطة** — بيطبّق بس المقترحات المُجمَع عليها والمُصاغة صح، `keep_open` افتراضياً، والرفض **بيسيب النقاش مفتوح** (fail-safe، مبيزيّفش تقارب).
2. **الاتفاق ≠ الاكتمال ≠ needs-user ≠ blocked** كحالات آلية منفصلة (بيقتل كذبة "الوكلاء قالوا خلصنا").
3. **النتيجة تُخزَّن قبل الـ finalizer** والـ synthesis ممنوع يناقضها (`server/orchestrator.js:639-650`).

### 1.3 الآلية الفريدة بصدق
**بلاش تبالغ:** `convergence:"converged"` و`substantiveDelta` **لسه تقارير ذاتية من الـ LLM** — نموذجين متملّقين ممكن يقفلوا بند غلط بالإجماع ويعدّي. اللي المحرّك بيضمنه **أضيق وأمين**: *الوكلاء مايقدروش يمثّلوا* (مفيش قفل صامت، مفيش قفل من طرف واحد، مفيش prose بيغلب الحالة). سوّقه كده بالظبط، مش "المحرّك بيقرّر".

### 1.4 المستخدم والوظيفة
- **مين يتبنّاه:** تقاطع (اشتراك Claude Code **و** Codex) + Node 22 + تغييرات "مخيفة" = مهندس senior/staff أو صيانة كود حرج (auth، migrations، أمان). + متبنّون عرب (تكافؤ ar/en مخدوم ضعيف في السوق).
- **التكلفة:** ~7 نداءات، **4-10× توكنز** و**3-5× وقت** مقابل وكيل واحد. يستاهل للـ 10% المخيفة، مش للشغل اليومي → **أداة episodic** (احتفاظها صعب — بتتشال بين الأزمات).
- **ضعف في الادعاء:** التنفيذ **غير متماثل** — Codex بس اللي بينفّذ، Claude review-only (`server/providers/registry.js:44`) → دايماً "Codex يكتب، Claude يراجع"، وده بيضعّف ادعاء التنوّع في **الكتابة**.

### 1.5 استراتيجية الأوبن سورس (الأهم)
- **التموضع اللي بيعيش:** *"بوابة مراجعة محلية للوكلاء: واحد يكتب في clone مؤقت، منافس يراجع read-only، سجل حتمي بيسجّل المتّفق والمفتوح، ومفيش حاجة بتلمس الريبو/الإيميل/GitHub قبل ما تضغط accept."* اقُد بـ **عدم-الثقة-في-الوكلاء**. الـ hook الجاهز في كودك: **"Agents can't close an issue by omission."**
- **الحركة الاستراتيجية: استخرج البروتوكول.** عقد `<agent-control>` v2 + `convergence.js` (421 سطر، صفر تبعيات، دوال نقية) = **spec + library** جاهزة. انشرها كبروتوكول مستقل + Codebate كتطبيق مرجعي. **التطبيقات بتتبلع من الـ vendors؛ البروتوكولات بتتبنّى من الـ frameworks** — ده النسخة الوحيدة اللي ليها إرث دائم.
- **مايكونش:** ❌ framework (N-agents بيدمّر دلالة الإجماع)، ❌ workbench موصلات (اقطع Gmail/Supabase لـ `examples/`)، ❌ IDE/سحابة/CI، وقلّل شل Electron.
- **الاحتمالات بصدق:** عمر مجتمع دائم كـ **تطبيق ~10-20%** (شرط الاشتراكين + خطر الابتلاع). كـ **بروتوكول + تطبيق مرجعي: أحسن بكتير**. (والاسم "Codebate" **عام وصعب البحث عنه**.)

### 1.6 أكبر خطر + أعلى رافعة
- **#1 خطر:** **قيمة غير مُثبَتة.** المهندس هيقول "أقدر أكتب *'يا Codex راجع خطة Claude واذكر الخلافات'* ببلاش — ليه سيرفر واشتراكين؟" ومفيش في الريبو إجابة.
- **#1 رافعة:** **اشحن دليل مرئي:**
  - **mini-eval** قابل للتكرار (N تغييرات فيها عيوب معروفة، 3 أذرع: مراجعة ذاتية / prompt ساذج / تدفق Codebate الكامل، بتقيس معدل مسك العيوب + معدل "الاتفاق الكاذب" + التوكنز — حتى N=25 بأرقام أمينة أقوى من كل التوثيق).
  - **GIF 90 ثانية** فوق README: وكيلان بيختلفوا، السجل بيثبّت `disagreement/open`، وكيل بيحاول يسكت عنه، والمحرّك **بيرفض التقارب** → كارت القرار البشري.

## المحور 2 — المعمار والتماسك

### 2.1 نقاط القوة
- **الطبقات حقيقية ومطبَّقة:** `convergence.js` **صفر imports** (مكتبة دوال نقية)؛ `run-state.js` آلة حالة صغيرة بمطالبة طرفية one-shot؛ الـ adapters بتستورد `process.js`/`logger.js` بس (مفيش معرفة بالجلسة)؛ الواجهة بتستهلك catalog المزوّدين ديناميكياً (إضافة مزوّد بدون تعديل HTML).
- **أفضل قرار تصميمي في الريبو:** النتيجة تُخزَّن **قبل** ما الـ finalizer يحكيها.

### 2.2 الـ god-modules (أكبر دين معماري)

| الملف | الحجم | المشكلة |
|---|---|---|
| **`server/worktree.js`** | 1011 سطر، ~30 export | **god-module حقيقي:** بيخبّي **بروتوكول WAL يدوي حول index-lock بتاع git** (`:530-868`، بـ SHA-256 وربط dev/ino/birthtime) جنب helpers تافهة زي `isGitRepo()`. **أعلى كود خطورة في الريبو** (بيعدّل الـ index والـ worktree **الحقيقيين**) جنب تنسيق الـ diff. يتقسم لـ: clone-lifecycle / acceptance / index-WAL+recovery / cleanup-GC. |
| **`public/app.js`** | 2280 سطر، ~30 global | **monolith منضبط** (منطق السباق مستخرَج لموديولات مختبَرة، epoch guards، render entry واحد) — بس عند **السقف**؛ 2000 سطر مغطّيين بـ**smoke test واحد للـ layout**. |
| **`server/index.js`** | 619 سطر | بيخلط **routing مع business logic** (`ghClone`، `listDirs`، `updateAgent` جوه الراوتر) + سلسلة `if` يدوية ~45 فرع حيث الترتيب load-bearing. |

**الحكم على "بدون framework":** مش مشكلة **دلوقتي** بسبب الانضباط، بس العلاج **مش React** — كمّل نمط الاستخراج (آلة حالة `running`/`currentRunId`/`liveAgents` هي المرشّح الجاي).

## المحور 3 — الموثوقية والاعتمادية

### 3.1 اللي اتصلّح فعلاً (موجود ومختبَر)
run-ID terminal fencing، stalled-stop finalize بتحرير idempotent، startup reconciliation ورا 503، schema migration بنسخ احتياطي + سجلات recovery، runtime lock بيرفض الثقة في PID حي بعد كراش. **✅ التصليب اللي عملناه حقيقي.**

### 3.2 الفجوات المتبقية الحقيقية (مرتّبة بالخطورة)

| # | الفجوة | الخطورة | الدليل |
|---|---|---|---|
| 1 | **SSE مفيش replay + العميل مابيزامنش بعد reconnect** — `onopen` بيقلب البادج بس، مابينده `loadSession()`. نوم اللابتوب/انقطاع بيبلع `run_complete` → الواجهة عالقة "running" والمدخلات مقفولة | 🔴 **أعلى باج مُحتمَل للمستخدم** | `server/index.js:182-187`, `public/app.js:792` — **إصلاح سطر واحد** |
| 2 | **Stop التنفيذ مابيوقفش الكاتب** — `git()` مابيمرّرش `registerChild` (Stop أثناء الـ clone مابيقتلش حاجة)، ومفيش فحص إلغاء بين `createWorktree` و`provider.run` → الـ executor بيكمّل. و`stopExec` بيرجّع `{stopped:true}` كذباً | 🔴 **عيب في أداة أمان** لمنتج شعاره "المستخدم مسيطر دايماً" | `server/exec-orchestrator.js:29-35`, `server/executor.js:19-27`, `server/worktree.js:105-123` |
| 3 | **الكتابة مفيش fsync** — atomic ضد كراش العملية، مش ضد قطع الكهرباء. الحالات الطرفية (accepted commit، `blocked_secret`، connector `executing_unknown`) هي بالظبط اللي الـ recovery بيعتمد عليها | 🟠 major | `server/store.js:206-231` (بينما `runtime-lock.js:95` بيعمل `handle.sync()`) |
| 4 | **runtime lock: liveness الـ heartbeat = liveness الـ event-loop** — عملية عالقة في شغل sync >30s وأطفالها بيكتبوا، سيرفر تاني ياخد القفل (نافذة محدودة self-healed). + يعتمد على mtime/ساعة واحدة → **بيتكسر على مجلد متزامن (OneDrive افتراضي على ويندوز)** ومفيش تحذير | 🟠 major | `server/runtime-lock.js:111-140`, `:199-216` (وino=0 على بعض volumes ويندوز) |
| 5 | **سجلات audit القراءة عالقة "running" بعد كراش مابتتصالحش** عند البدء | 🟡 minor | `server/connectors/service.js:86-101` |

### 3.3 صفر تبعيات — قوة كبيرة بتكلفة مسمّاة
**قوة:** سطح supply-chain ~صفر، `--ignore-scripts`، action SHAs مثبّتة. **تكلفة:** المشروع بيملك نسخ يدوية من (markdown renderer، SSE، migration engine، WAL، قتل شجرة العمليات بـ PowerShell WMI walk `server/process.js:444-464` — snapshot-then-kill سباق بطبيعته)، كل واحدة صغيرة بس محتاجة **مراجعة متخصّصة للأبد ومفيش upstream fixes**.

## المحور 4 — الأمان والخصوصية (أقوى محور)

### 4.1 النموذج (ممتاز)
توكن 256-bit لكل تشغيل، cookie **HttpOnly+SameSite=Strict** (بيغطي الـ EventSource اللي مايقدرش يبعت headers)، فحوص **Host+Origin بمنفذ مطابق** (بيقفل DNS-rebinding/CSRF)، **CSP قوي `script-src 'self'`** (بيمنع inline scripts و`javascript:` — backstop حقيقي للـ innerHTML)، `timingSafeEqual` بحارس byte-length، allowlists للبيئة، عزل الاعتمادات عن بيئة الـ subprocess. (`server/security.js`)

### 4.2 نموذج التنفيذ المعزول (متطور)
clone معزول بالكائنات (`--no-local --no-checkout`)، **ربط الشجرة الثابتة** (`reviewedTree` treeSha)، **إعادة فحص أسرار fail-closed عند القبول**، دفاع commondir/alternates/symlink، دفاع أسماء الفروع/المسارات، **CAS على refs مع index.lock حقيقي**، commits الـ executor بتتجمّع بـ `git reset --mixed`، commit واحد بـ `commit-tree`، **مفيش `full` mode**، والقبول ملك Codebate مش الـ executor. (`server/worktree.js`, `EXECUTION.md`)

### 4.3 المخاطر المتبقية
- **TOCTOU في الـ spawn** — موثّق كمقبول (`server/process.js:192-198`).
- **فحص الأسرار heuristic** ("guard مش guarantee" `server/secret-scan.js:1-5`) — regexes؛ صيغة سر جديدة ممكن تعدّي. الضمان بيقع على ربط الشجرة الثابتة + إعادة الفحص.
- **capability-router بالـ regex** (عربي/إنجليزي) — بس **بيفشل بأمان** (الـ discussion read-only بغض النظر). (`server/capability-router.js`)
- **مفيش تدقيق أمني خارجي رسمي** بعد.

## المحور 6 — تجربة المستخدم والـ i18n والوصولية

- **غرفة القرار:** phase **مشتقّة** مش يدوية، كارت لكل مزوّد، outcome رسمي واحد، tracker مراحل **monotonic** بأوقات من مصادر أمينة بس، **SSE = حالة حيّة مؤقتة / الجلسة المُعاد جلبها = الحقيقة**. تصميم أمين.
- **i18n قوي جداً:** قواعد جمع عربي كاملة (`public/i18n-core.js:116`)، Intl formatters، CSS logical properties، bidi دقيق.
- **🔴 الباج الحقيقي (الاتنين مستشارين أكّدوه):** الأوركستريتور بيخزّن **نصوص عربي hardcoded** في الـ transcripts (`server/orchestrator.js:164-191`, `server/exec-orchestrator.js:216-227`) بينما باقي الرسائل إنجليزي → **transcripts مختلطة "مكسورة" للمستخدم الإنجليزي**. + اللغة الافتراضية **عربي بغض النظر عن لغة المتصفح** (`public/app.js:2255`). **P0 قبل الإطلاق.**
- **الوصولية:** قوية (focus traps، ARIA tabs، live regions، reduced-motion) بس فجوات: **تباين `--faint:#7a8796`** على أسطح فاتحة غالباً بيسقط تحت 4.5:1 لنص صغير، `window.confirm/alert` بتتخطى الـ focus المُدار، **مفيش skip-link**.

## ✅ التوصيات المرتّبة قبل الإطلاق

1. **صلّح "Stop اللي مابيوقفش"** — مرّر `registerChild` لـ `git()`، افحص الإلغاء قبل spawn الـ executor، وادّي `stopExec` معاملة `stopRun` (مع تست `runExecuteAndReview` الناقص).
2. **خلّي العميل يشفي نفسه** — `loadSession()` في `source.onopen` (+ event IDs + replay ring صغير).
3. **قرّر اللغة** — كشف locale المتصفح + توقّف تخزين نصوص سيرفر بلغة واحدة.
4. **افتح ممر المساهمين** — إمّا CI يفرض الـ gate، إمّا checklist يدوي موثّق + اقتراح تركيب الـ hooks.
5. **اشحن دليل القيمة** — mini-eval بأرقام أمينة + GIF 90 ثانية (أعلى رافعة تسويقية).
6. **بعد الإطلاق مباشرة:** قسّم `worktree.js`، JSDoc+`checkJs`، fsync للحالات الطرفية، استخرج البروتوكول كـ package مستقل.

## 🏁 الحكم النهائي

**الهندسة جاهزة للإطلاق — والـ run/acceptance/recovery أدق من أدوات مموَّلة في نفس المجال.** لكن المشروع **novel بآلية واحدة عميقة، مش كفئة**. النجاح الدائم مشروط بـ:
- **(أ)** إعادة تموضع حوالين "بوابة المراجعة الأمينة" مش "سحر المناظرة".
- **(ب)** إثبات القيمة بأرقام.
- **(ج)** استخراج البروتوكول كإرث يعيش حتى لو الـ vendors ابتلعوا التطبيق.

صلّح الـ 3 الحمرا (Stop / SSE / اللغة) وهتبقى أقوى إطلاق أوبن سورس في المساحة دي.
