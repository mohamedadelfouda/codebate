# Codebate

**خلّي وكلاء البرمجة يناقشوا ويراجعوا بعض — وإنت تفضل ماسك القرار.**

Codebate مساحة عمل محلية بتجمع Claude Code وCodex وCursor على نفس المهمة، وتديهم نفس السياق المحدود من مشروعك، وتخلّيهم يراجعوا ويعترضوا على اقتراحات بعض، وفي النهاية تعرضلك نتيجة واضحة تقدر تفهمها وتراجعها.

Codebate مش مجرد أكتر من شباك شات جنب بعض.
هو بيتابع كل Agent اقترح إيه، اتفقوا على إيه فعلًا، إيه اللي لسه مختلفين عليه، والجلسة انتهت ليه. ولو المهمة فيها تعديل كود، التنفيذ بيحصل داخل نسخة مؤقتة من الريبو، ومفيش حاجة بتدخل مشروعك الحقيقي غير بعد موافقتك الصريحة.

## ليه بنيت Codebate؟

طريقة شغلي كانت بتمشي كده:

1. أطلب من Claude يعمل خطة.
2. أنسخ الخطة إلى Codex علشان يراجعها.
3. أرجع اعتراضات Codex إلى Claude.
4. أكرر العملية كذا مرة.
5. أحاول أفتكر أنهي نسخة هي الأخيرة، وإيه اللي الاتنين اتفقوا عليه فعلًا.

الـAgents كانوا مفيدين، لكن أنا كنت بشتغل كوسيط بين الأدوات.
Codebate بيحوّل النقل اليدوي ده إلى Workflow منظم ومحلي.
كل Agent بيشوف نفس الأدلة، ويرد على باقي الـAgents، وفي النهاية بيفضل عندك سجل واضح للاقتراحات والخلافات والقرارات والخطوات الجاية.

## Codebate بيعمل إيه؟

### نفس الأدلة لكل الوكلاء

كل Agent مشارك بيشتغل من نفس السياق المحدود بدل ما كل واحد ياخد Prompt مختلف منسوخ يدويًا.
ملفات المشروع مش بتكون متاحة قبل ما توثّق المشروع بنفسك.

### ثلاثة أوضاع للمحادثة

**Chat**
ابعت نفس السؤال لأكثر من Agent وقارن إجاباتهم المستقلة.

**Collaboration**
الـAgents يشتغلوا على المهمة في جولات، ويحسنوا اقتراحات بعض، ويحاولوا يوصلوا إلى نتيجة مشتركة.

**Debate**
Agentين ياخدوا اتجاهين مختلفين، يختبروا الافتراضات، ويظهروا نقاط الخلاف اللي لسه متحلتش.

### اتفاق حقيقي مش مجرد خاتمة مقنعة

Codebate مش بيعتبر إن الـAgents اتفقوا لمجرد إن آخر رد مكتوب بثقة.
هو بيتابع بشكل منفصل:

* اقتراحات العمل الجديدة؛
* البنود المفتوحة المعتمدة؛
* البنود اللي اتحلت أو اندمجت؛
* القرارات اللي لسه محتاجة منك تدخل؛
* الخطوة المطلوبة من المستخدم؛
* السبب الحقيقي إن الجلسة كملت أو وقفت.

الملخص النهائي بيشرح النتيجة، لكنه مش بيغيّر حالة الجلسة الرسمية من نفسه.

### تنفيذ → مراجعة → قرار

في المهام اللي فيها تعديل كود:

1. Agent منفّذ يشتغل داخل نسخة مؤقتة من الريبو.
2. Codebate يحفظ شجرة الملفات الناتجة والـdiff المحدود.
3. Agent مختلف يراجع نفس النتيجة من غير ما يعدلها.
4. إنت تراجع الاقتراح والتنفيذ والمراجعة.
5. موافقتك فقط هي اللي تسمح للنتيجة اللي اتراجعت إنها تدخل الريبو الحقيقي.

أي تنفيذ ترفضه يفضل خارج مشروعك.

### موافقة بشرية قبل أي تأثير خارجي

الحاجات زي فتح Pull Request أو إرسال إيميل أو تعديل بيانات متصلة بتفضل مجرد اقتراحات لحد ما توافق عليها.
Codebate بيساعدك تاخد القرار، مش بياخد القرار بدلًا منك في الخفاء.

## أدوار الأدوات

Codebate بيستخدم أدوات الـCLI والاشتراكات الموجودة عندك بالفعل.

| الأداة      | Chat | Collaboration | Debate | Review | تنفيذ معزول |
| ----------- | :--: | :-----------: | :----: | :----: | :---------: |
| Claude Code |   ✓  |       ✓       |    ✓   |    ✓   |      —      |
| Codex CLI   |   ✓  |       ✓       |    ✓   |    ✓   |      ✓      |
| Cursor CLI  |   ✓  |       ✓       |    ✓   |    ✓   |      —      |

التنفيذ بيتاح فقط للأداة اللي Codebate يقدر يفرض عليها حدود التشغيل المطلوبة. مجرد إن الأداة تقدر تعدل ملفات مش معناه إنها تتسجل كمنفّذ.

## التشغيل السريع

### المتطلبات

* Node.js 22 أو أحدث
* Git
* أداتين على الأقل من أدوات البرمجة المدعومة، متثبتين ومسجل فيهم دخول:
  * Claude Code
  * Codex CLI
  * Cursor CLI
* GitHub CLI (`gh`) فقط لو هتستخدم عمليات مرتبطة بـGitHub

### التشغيل من npm

```bash
npx codebate
```

Codebate بيشغّل سيرفر محلي ويفتح مساحة العمل في المتصفح.
الـPrompts وردود الـAgents والجلسات والقرارات بتفضل محفوظة محليًا. Codebate مش بيمرر استخدام النماذج على Backend سحابي تابع له.

### التشغيل من السورس

```bash
git clone https://github.com/mohamedadelfouda/codebate.git
cd codebate
corepack enable
pnpm install --prod --frozen-lockfile --ignore-scripts
pnpm start
```

العنوان الافتراضي:

```text
http://127.0.0.1:3210
```

## نموذج الأمان

Codebate مبني على حدود تشغيل واضحة، مش على الثقة في تعليمات مكتوبة داخل الـPrompt فقط.

* ملفات المشروع مش بتتشارك قبل ما توثّق المشروع.
* الملفات التنفيذية الخاصة بالـProviders بيتحفظ لها بصمة.
* التخطيط والمراجعة بيكونوا قراءة فقط.
* بيئة الـAgents بتستخدم قائمة متغيرات مسموح بها، ومبتورثش مفاتيح عشوائية.
* حجم المخرجات ومدة التشغيل والأدلة المحملة وبيانات الجلسات كلها محدودة.
* التنفيذ بيحصل داخل نسخ مؤقتة لها Git objects وإعدادات مستقلة.
* نفس شجرة الملفات اللي اتراجعت بيتم التحقق منها مرة ثانية قبل القبول.
* بيتم فحص الأسرار قبل إدخال التغييرات إلى المشروع.
* مفيش Executor بصلاحيات مفتوحة أو نشر متوافق عليه مقدمًا.
* بيانات دخول الموصّلات بتفضل على الجهاز ومبتدخلش الـPrompts.
* أي كتابة خارجية محتاجة موافقة صريحة من المستخدم.

النسخة المؤقتة من Git مش Sandbox كامل لنظام التشغيل. راجع نموذج التهديد الكامل داخل [SECURITY.md](SECURITY.md).

## محلي من الأساس

Codebate مش بديل لـClaude Code أو Codex أو Cursor.
هو بينظم الأدوات اللي إنت أصلًا بتستخدمها.
الـPrompts والأجزاء المحدودة من مشروعك بتتبعت عن طريق أدوات الـCLI الرسمية الخاصة بالـProviders. مفيش حساب Codebate، ولا Agent Runtime مستضاف، ولا اشتراك نماذج إضافي.

## نطاق المشروع

Codebate مركز على مشكلة واحدة:

> مساعدة المطور إنه يطلع شغل أكثر موثوقية من أكثر من Coding Agent، من غير ما يفقد الرؤية أو التحكم.

المشروع مش بيحاول يبقى شركة برمجيات ذاتية التشغيل، ولا Agent خفي يشتغل في الخلفية، ولا منصة Workflow عامة لكل شيء.
الأولوية هي إن التعاون والمراجعة والتنفيذ والموافقة يبقوا واضحين وقابلين للفهم والثقة.
